# 车队调控算法使用手册

本文档总结了接入车队调控算法后的新增功能、调度机制和前端可视化增强，便于快速了解系统能力和使用方式。

## 1. 核心能力概览

### 1.1 调度与路由
- 支持基于图结构的最短路规划，车辆沿路径连续行驶而非节点跳变。
- 支持两种调度策略：
  - 最近任务优先（nearest）
  - 最大任务优先（max_weight）
- 支持车辆电量约束、充电站排队与负荷评估，自动选择更优充电站。

### 1.2 任务执行规则
- 所有货物必须从中央仓库发出。
- 车辆执行任务前若不在仓库，会先回仓装货（或先去充电）。
- 单车模式下支持超载任务自动拆分为多次往返运输，直到任务完成。
- 多车协同模式下可拆分任务重量并行完成。

### 1.3 评分与超时
- 任务完成得分与完成时间、路径距离相关。
- 超时未完成会扣分，超时惩罚会累积。

## 2. 前端可视化增强

### 2.1 地图渲染
- 车辆实时移动可视化，不再是节点间跳变。
- 充电站节点绿色高亮，仓库红色高亮，任务节点橙色高亮。
- 鼠标悬停节点显示节点编号和类型信息。

### 2.2 车辆悬浮信息
- 鼠标悬停车辆显示任务信息、状态、电量、载重等。
- 车辆编号在地图上显示，重叠时标签自动错位排列。

### 2.3 任务列表面板
- 左侧任务栏显示当前任务列表。
- 点击任务展开详细信息：重量、生成时间、目标节点、执行车辆编号。
- 展开信息内提供任务进度条（基于车辆位置和载重，平滑变化）。
- 点击任务可在地图中高亮目标节点与执行车辆。

## 3. 任务生成机制

- 任务生成速度已调整为“等待-递增”模式：
  - 模拟时间前 30 秒，生成概率为 0。
  - 30 秒后，每秒增加 2% 生成概率。
  - 一旦生成新任务，概率重置并重新等待 30 秒。
- 充电站不会作为任务终点。

## 4. 使用与配置要点

### 4.1 启动与访问
- 启动后端：运行 `app.py`
- 访问页面：打开 `http://127.0.0.1:5000/`

### 4.2 调整车队与调度
- 车辆数量：在 `app.py` 中修改车队初始化数量。
- 调度策略：在 `app.py` 中设置 `strategy` 为 `nearest` 或 `max_weight`。
- 多车协同：在 `app.py` 中设置 `allow_multi_vehicle`。

### 4.3 调整仿真节奏
- 时间倍率：`app.py` 中 `time_scale` 控制仿真时间加速。
- 前端刷新率：`templates/index.html` 中刷新间隔控制 UI 流畅度。

## 5. 关键实现文件

- 车队调度核心：`Fleet_Controller.py`
- 后端服务与状态接口：`app.py`
- 任务生成逻辑：`Task_Manager.py`
- 前端可视化：`templates/index.html`

## 6. 建议的后续扩展

- 将任务进度、车辆任务链等信息加入 API 输出，便于统计与报表。
- 接入强化学习或元启发式算法，替换现有贪心策略。
- 增加任务优先级或服务等级（SLA）约束。
