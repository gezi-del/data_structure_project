# 数据结构小组作业——基于图网络的物流调度系统
- 数据结构小组作业，主要内容为**图网络上的物流调度系统**
- 组员分工：
  - 地图生成：李俊德、葛子轩
  - 调度策略：罗祯祥
  - 任务信息生产：王浩源


本项目为数据结构小组作业，主要内容为“图网络上的物流调度系统”。系统以 Python 实现，并辅以少量前端 HTML 页面，涵盖了与物流调度相关的车辆、任务、图结构管理及调度算法实现。项目既适合作为数据结构课程项目，也可作为类似场景的参考实现。

---
## 目录结构及主要文件说明

- **Fleet_Controller.py**  
  车队调度核心模块。实现了车辆资源的统一调度、路径分配、状态管理等与车队全局调度策略相关的业务逻辑。

- **Graph.py**  
  图结构定义与操作模块。用于描述物流网络中的节点（如仓库、配送点）与边（路线），实现了图的基本管理和路径算法（如最短路径搜索等）。

- **Task.py**  
  任务实体类，定义了派单、任务属性以及与任务相关的基本方法（如状态流转、分配等）。

- **Task_Manager.py**  
  任务管理模块。负责统一管理所有任务（接收、新建、分派、状态管理等），并协调各任务的生命周期与调度。

- **Vehicle.py**  
  车辆实体类。定义了每辆车的基本属性和方法，包括载重、位置、可用性等，以及与任务的关联。

- **app.py**  
  系统主入口（后端/服务端），通常以 Flask/FastAPI 等方式启动，负责运行整个物流调度系统的后端服务。  
  运行方式：  
  ```bash
  python app.py
  ```

- **main.py**  
  快速测试或脚本入口。可用于执行某些脚本化操作或简单测试。

- **templates/**  
  前端 HTML 模板目录。存放 Web 服务页面所用的 html 文件，用于可视化展示系统状态或与用户交互。

- **README.md**  
  项目说明文档（本文件）。

- **LICENSE**  
  开源许可证，默认为 MIT 协议。

- **Vehicle_使用手册.md**  
  车辆模块的使用说明文档，详细介绍了如何创建和管理车辆对象及相关接口。

- **车队调控算法_使用手册.md**  
  介绍系统中车队调度/调控算法的实现原理及使用指南。

- **运行app.py就好了.txt**  
  说明性提示，只需运行 `app.py` 即可启动系统。
